# Excel转Shapefile转换器 - 实施规划

## 项目概述
构建一个Java 17工具，将Excel文件转换为Shapefile格式，具有智能表头检测、坐标列匹配、多行表头合并、合并单元格处理，以及多sheet工作簿支持功能。每个sheet工作簿代表一种数据类型，自动生成对应的Shapefile文件。

## 需求总结

### 核心功能需求

#### 1. 表头检测
- **自动检测表头行** - 从Excel文件中自动识别表头行（支持多行表头和合并单元格）
  - 表头可能占据**多行**，需要智能合并列名
  - 正确处理Excel中的**合并单元格**
  - 返回完整的、去重的列名数组
  - 处理边缘情况：空行、全数字行等

#### 2. 数据提取
- **自动检测数据结束** - 遇到空行时停止读取数据
- **支持中英文列名** - 支持中文（经度/纬度）和英文（longitude/latitude, X/Y等）列名

#### 3. 坐标列识别与几何类型
- **自动匹配坐标列** - 自动识别两种数据模式，并根据模式生成相应几何类型：

  **单点模式**：
  - 分开的X/经度和Y/纬度列各1个 → 生成**Point几何**
  - 例如：经度、纬度 分别在两列

  **起止点模式**：
  - 四字段模式（起点经度、起点纬度、终点经度、终点纬度）→ 生成**LineString几何**
  - **关键点**：部分点数据可能为起止点四字段展示形式，需要自动识别这种模式并转换为LineString

#### 4. 多工作簿处理
- **支持Excel中的多个sheet工作簿** - 每个sheet代表一种数据类型
  - 循环处理每个sheet工作簿，各自生成一个独立的Shapefile文件
  - 每个sheet的输出文件名基于sheet名称
  - 根据各sheet数据的坐标模式自动选择几何类型（Point或LineString）
  - 确保每个sheet的数据在生成shpfile时正确识别为单点或起止点模式

### 技术栈
- **Maven构建系统** - 使用Maven进行依赖管理和构建

## 核心算法

### 1. 表头检测算法（支持多行表头和合并单元格）
**多行表头处理流程：**
1. 扫描前N行（可配置，默认10行），查找表头起始行
2. 对每行评分：
   - **+10分** 如果行索引为0
   - **+5分** 每个文本单元格
   - **-3分** 每个数字单元格
   - **+8分** 如果没有重复值
3. 选择得分最高的行作为表头起始行
4. **多行表头合并**：
   - 检测表头起始行下是否还有连续的非数字行（表示有多行表头）
   - 将多行表头的单元格值合并（用"_"连接或堆叠）
   - 解决合并单元格问题：对于Excel中的合并单元格，只在第一行/列显示值，其他位置为空
   - 智能填充合并单元格的空白处（使用前一行/列的值或特殊标记）
5. 返回完整的列名数组（已处理多行和合并情况）

### 2. 数据模式识别与坐标列匹配（支持两种数据模式）

**第一步：数据模式识别**
- 扫描所有列名，检测是否存在起止点四字段模式的特征关键字
- **单点模式**：只有1对X/Y列（经度/纬度），生成Point几何
- **起止点模式**：有4个坐标列，包含起点和终点关键字，生成LineString几何

**关键特征识别：**
- 起点关键字：含有"起点"、"start"、"起始"、"from"等
- 终点关键字：含有"终点"、"end"、"结束"、"to"等
- 如果发现起/终点关键字，则判定为起止点模式（LineString）
- 否则为单点模式（Point）

**第二步：坐标列匹配算法

**三层优先级模式匹配：**

**优先级1 - 精确匹配：**
- 单点X: "经度", "longitude", "lon", "lng", "x"
- 单点Y: "纬度", "latitude", "lat", "y"
- 起点X: "起点经度", "起点lon", "起点x", "start_lon", "start_x"
- 起点Y: "起点纬度", "起点lat", "起点y", "start_lat", "start_y"
- 终点X: "终点经度", "终点lon", "终点x", "end_lon", "end_x"
- 终点Y: "终点纬度", "终点lat", "终点y", "end_lat", "end_y"

**优先级2 - 包含匹配：**
- X: "x坐标", "x_coord", "x coord", "东经"
- Y: "y坐标", "y_coord", "y coord", "北纬"
- 起点: "起点", "start", "起始", "from"
- 终点: "终点", "end", "结束", "to"

**优先级3 - 正则表达式：**
- X: `.*[xX].*|.*lon.*|.*经.*`
- Y: `.*[yY].*|.*lat.*|.*纬.*`
- 起点: `.*起.*点.*|.*start.*|.*from.*`
- 终点: `.*终.*点.*|.*end.*|.*to.*`

所有匹配均不区分大小写。

### 3. 数据提取算法
- 从表头后一行开始读取
- 对每一行：检查是否完全为空 → 停止
- 将所有单元格值提取到 Map<String, Object>
- 解析X/Y坐标并验证范围
- 跳过无效行（可配置：严格模式快速失败）

## 实施阶段

### 阶段1：项目设置
**需要创建的文件：**
- `pom.xml` - Maven配置和依赖
- 目录结构：`src/main/java/com/dataconversion/`
- 包目录：reader, analyzer, writer, model, util

**依赖项：**
- Apache POI 5.2.5 (poi-ooxml)
- GeoTools 30.1 (gt-shapefile, gt-epsg-hsql)
- SLF4J 2.0.9 + Logback 1.4.14
- JUnit Jupiter 5.10.1

**注意：** 需要添加OSGeo仓库以获取GeoTools

### 阶段2：模型类
**需要创建的文件：**
1. `model/ConversionConfig.java`
   - inputFilePath, outputFilePath
   - sheetIndex (默认: 0)
   - headerRowIndex (null = 自动检测)
   - xColumnName, yColumnName (null = 自动检测)
   - crs (默认: "EPSG:4326")
   - strictMode, skipInvalidRows

2. `model/ColumnMapping.java`
   - xColumnIndex, yColumnIndex
   - columnNames数组
   - Excel列到shapefile属性的映射

3. `model/ExcelData.java`
   - 数据行列表
   - 表头信息
   - 元数据（行数等）

4. `util/ConversionException.java`
   - 基础异常类
   - 子类：InputValidationException, HeaderDetectionException, CoordinateMatchingException

### 阶段3：Excel读取组件
**需要创建的文件：**
1. `reader/ExcelReader.java`
   - 使用Apache POI打开工作簿 (XSSFWorkbook)
   - 处理.xlsx格式
   - 按索引选择工作表
   - 向其他组件提供Sheet对象

2. `reader/HeaderDetector.java`
   - 实现评分算法
   - 扫描前N行（可配置，默认10）
   - 返回包含行索引和列名的HeaderInfo
   - 处理边缘情况：全数字行、空行

3. `reader/DataExtractor.java`
   - 从表头后开始读取行
   - 遇到第一个完全空行时停止
   - 将Excel单元格转换为Java类型
   - 处理单元格类型检测（数字、字符串、日期）
   - 返回 List<Map<String, Object>>

### 阶段4：坐标分析组件
**需要创建的文件：**
1. `util/CoordinatePatterns.java`
   - 定义X和Y列的模式数组
   - 精确、包含和正则表达式模式
   - 支持中英文名称

2. `analyzer/CoordinateColumnMatcher.java`
   - 实现三层匹配算法
   - 不区分大小写的比较
   - 返回包含索引的ColumnMapping
   - 如果未找到则抛出带有帮助信息的异常

3. `analyzer/DataValidator.java`
   - 验证坐标值为数字
   - 检查范围：经度 [-180, 180]，纬度 [-90, 90]
   - 返回验证结果
   - 支持严格模式（快速失败）和宽松模式（跳过无效）

### 阶段5：Shapefile写入组件
**需要创建的文件：**
1. `writer/SchemaBuilder.java`
   - 构建GeoTools SimpleFeatureType
   - 添加Point几何字段
   - 为每个Excel列添加属性字段
   - 清理属性名称（shapefile的10字符限制）
   - 在要素类型上设置CRS

2. `writer/ShapefileWriter.java`
   - 创建ShapefileDataStore
   - 创建SimpleFeatureBuilder
   - 对每个数据行：
     - 从X, Y坐标创建Point几何
     - 创建包含所有属性的SimpleFeature
     - 添加到要素集合
   - 将要素写入shapefile
   - 生成.shp, .shx, .dbf, .prj文件

### 阶段6：主协调器
**需要创建的文件：**
1. `ExcelToShapefileConverter.java`
   - 带有main()方法的主入口
   - 解析命令行参数
   - 创建ConversionConfig
   - 协调工作流程：
     1. 打开Excel文件 (ExcelReader)
     2. 检测表头 (HeaderDetector)
     3. 匹配坐标列 (CoordinateColumnMatcher)
     4. 提取数据 (DataExtractor)
     5. 验证数据 (DataValidator)
     6. 构建模式 (SchemaBuilder)
     7. 写入shapefile (ShapefileWriter)
   - 处理异常和日志
   - 返回转换摘要

## 主要工作流程

### 多工作簿处理流程

```
1. 配置与验证
   ├─ 解析参数或配置文件
   ├─ 验证输入文件存在
   └─ 设置默认值 (CRS: EPSG:4326)

2. 打开Excel工作簿
   ├─ 使用Apache POI加载工作簿
   ├─ 获取所有工作表（Sheet）
   └─ 遍历每个工作表进行转换

3. 【循环处理每个Sheet】对于每个Sheet工作表：

   3.1 表头检测与数据提取
       ├─ 检测多行表头（支持合并单元格）
       ├─ 合并多行表头列名
       ├─ 提取列名数组
       └─ 读取该Sheet的所有数据行（遇到空行停止）

   3.2 数据模式识别
       ├─ 扫描该Sheet的所有列名
       ├─ 识别是否为起止点四字段模式
       ├─ 判定几何类型：Point 或 LineString
       └─ 匹配对应的坐标列（单点模式或起止点模式）

   3.3 数据验证
       ├─ 验证坐标值为数字
       ├─ 检查坐标范围（经度[-180,180], 纬度[-90,90]）
       ├─ 对起止点模式的4个坐标列进行验证
       └─ 跳过或标记无效行

   3.4 几何与属性构建
       ├─ 创建SimpleFeatureType（根据几何类型）
       ├─ 根据几何类型添加几何字段：
       │  ├─ Point模式：单个Point几何
       │  └─ LineString模式：从起点到终点的LineString
       ├─ 为每个Excel列添加属性字段
       └─ 设置CRS (EPSG:4326)

   3.5 Shapefile写入
       ├─ 生成输出文件名：基于Sheet名称
       ├─ 创建ShapefileDataStore
       ├─ 创建SimpleFeatureBuilder
       ├─ 对每个数据行构建SimpleFeature：
       │  ├─ Point模式：从X/Y创建Point几何
       │  └─ LineString模式：从起点和终点坐标创建LineString
       ├─ 添加所有属性值
       ├─ 将要素写入Shapefile
       └─ 生成文件：.shp, .shx, .dbf, .prj

   3.6 转换摘要
       ├─ 记录处理的行数
       ├─ 记录输出文件位置
       ├─ 记录几何类型（Point/LineString）
       └─ 记录任何警告或错误

4. 所有Sheet处理完成
   └─ 返回整体转换摘要（各Sheet处理情况）
```

### 关键特性说明

**多行表头与合并单元格处理：**
- 表头检测算法会自动识别多行表头并进行智能合并
- 对于Excel中的合并单元格，会自动填充空白处，确保每列都有完整的列名

**起止点四字段模式识别：**
- 如果表头中存在"起点"、"终点"等关键词，自动识别为LineString模式
- 此时需要4个坐标列：起点经度、起点纬度、终点经度、终点纬度
- 系统会自动从起点和终点坐标生成LineString几何对象

**每个Sheet独立转换：**
- 每个Sheet作为一个独立的数据类型进行处理
- 根据该Sheet的数据模式自动判断生成Point还是LineString几何
- 输出的Shapefile文件名基于Sheet名称
- 不同Sheet的数据不会混合，完全独立生成

## 错误处理

### 输入验证（快速失败）
- 文件未找到 → "Excel文件未找到: [路径]"
- 无效格式 → "不是有效的Excel文件"
- 空文件 → "Excel文件不包含数据"

### 检测错误（带指导的失败）
- 未检测到表头 → "无法检测表头行。使用 --header-row 指定"
- 未找到坐标列 → "无法找到坐标列。可用列: [列表]。使用 --x-column 和 --y-column"

### 数据验证（可配置）
- 无效坐标 → 记录警告，跳过行（默认）或失败（严格模式）
- 超出范围 → 记录带行号的警告
- 缺失值 → 跳过行，记录警告

## 配置选项

**命令行界面：**
```bash
# 基本用法
java -jar excel-to-shapefile.jar input.xlsx output.shp

# 带选项
java -jar excel-to-shapefile.jar \
  --input data.xlsx \
  --output result.shp \
  --header-row 1 \
  --x-column "经度" \
  --y-column "纬度" \
  --crs "EPSG:4326" \
  --strict
```

**ConversionConfig属性：**
- inputFilePath (必需)
- outputFilePath (必需)
- sheetIndex (默认: 0)
- headerRowIndex (null = 自动检测)
- xColumnName (null = 自动检测)
- yColumnName (null = 自动检测)
- crs (默认: "EPSG:4326")
- strictMode (默认: false)
- skipInvalidRows (默认: true)
